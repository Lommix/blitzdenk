Features

Features

Terminal UI

src/app.zig owns the TUI state: chat history, input editing, command palette, status bar, notifications, permission modals, passphrase prompts, tool previews, diff rendering, and streaming responses.

Agents and swarms

src/provider/agent.zig runs one agent. src/provider/swarm.zig manages many agents, permissions, broadcasts, active slots, cancellation, retries, and shared execution state.

Built-in agent types:

AgentPurpose
generalmain coding agent with write/edit/bash/read/tasks/ask/subagent tools
exploresearch and research agent with read/rg/skill/MCP/LSP/subagent tools

Lua can register more agent definitions with blitz.add_agent.

Providers

Provider-neutral chat and streaming types live in src/provider/adapter.zig. Provider implementations live in:

ModulePurpose
openai.zigOpenAI-compatible request serialization, completion, streaming parse
anthropic.zigAnthropic request serialization, completion, streaming parse
config.zigmodel/provider config, key env vars, reasoning effort
http.zigasync request pool used by provider calls

Supported provider kinds are openai, anthropic, and ollama.

Lua extension

Lua config can add tools, providers, agents, modes, commands, key binds, MCP servers, LSP servers, listeners, notifications, and queue actions.

See Lua API.

Tools

Built-in tools cover file read/write/edit, patch application, shell commands, ripgrep search, user questions, task tracking, subagents, SSH mode, skills, MCP, and LSP.

See Tools.

Permissions

Tools request permission through ToolContext.requestPerm. Permission payloads include shell commands, tool calls, ask prompts, and plan approvals. The app renders active permission requests and resolves them back into the swarm.

Compaction

src/provider/compact.zig estimates context size, starts summarization before the context limit is exceeded, polls compaction progress, and installs compacted history after completion.

Sessions and history

src/session.zig saves and loads session state. src/app.zig stores prompt history in prompt_history.json, keeps recent chat entries, and can queue messages for agents from Lua.

MCP and LSP

MCP servers are started from Lua config and their remote tools become Blitzdenk tools. LSP servers are also started from Lua and exposed through one lsp tool with operations for hover, definitions, references, document symbols, and workspace symbols.
